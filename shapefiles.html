Shapefiles

O pacote measurements permite converter latitude e longitude em radianos (graus, minutos, segundos) para decimal ou vice-versa.

Por exemplo o bairro do Rio Pequeno em So Paulo latitude e longitude decimal: -23.5665987,-46.7586146. Se quisssemos converter -23.5665987 para coordenadas em radianos faramos:

minutos = 60*0.5665987
segundos = 60*(minutos-33)
print(minutos)
## [1] 33.99592
print(segundos)
## [1] 59.75532

Assim -23.5665987 corresponderia a: -23 33 59.75532.

Usando o pacote measurements:

library(measurements)
conv_unit(x = -23.5665987, from = "dec_deg", to = "deg_min_sec")
## [1] "-23 33 59.7553199999966"

O inverso, converter -23 33 59.75532 para decimal:

23 + 33/60 + 59.75532/(60*60)
## [1] 23.5666

Ou usando a biblioteca:

conv_unit(x = "-23 33 59.75532", from = "deg_min_sec", to = "dec_deg")
## [1] "-23.5665987"

Para abertura do shapefile usamos o pacote rgdal. Como exemplo baixei um shapefile do site https://www.ibge.gov.br/geociencias/organizacao-do-territorio/malhas-territoriais/15774-malhas.html em Downloads, municipio_2020, Minas Gerais.

Os arquivos shp, dbf, shx e prj devem estar em um nico diretrio e com o mesmo nome em cada extenso. Exemplo, no diretrio MG_Municipios vamos ler os arquivos MG.shp, MG.dbf, MG.shx e MG.prj:

library(rgdal)
shapefile_MG <- readOGR(dsn = "data/shapefiles/MG_Municipios", layer = "MG")
## OGR data source with driver: ESRI Shapefile 
## Source: "/home/thiago/repos/r-notebooks/data/shapefiles/MG_Municipios", layer: "MG"
## with 853 features
## It has 4 fields

Lembrando que:

  • shp: geometria, isto , os polgonos do mapa
  • dbf: base de dados
  • shx: relaciona os arquivos shp e dbf
  • prj: sistema de projeo geogrfica

Para acessar os dados usamos o arroba e depois o nome da varivel:

head(shapefile_MG@data$NM_MUN)

sp = read.csv("data/sp.csv")

shapefile_SP <- merge(x = shapefile_SP,
